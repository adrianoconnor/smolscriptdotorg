roadmap

SmolScript is a mini programming laguage and runtime. It is based on simple Javascript, it features a tiny compiler and VM that and can be easily embeded in dotnet and Javascript applications.

Our VM provides some interactive debugging features, making it great for building interactive educational apps and, because it is based on Javascript, it is a great learning language.

It is not fast, it is not fancy, but it is very smol.

why?

There are lots of reasons why you would want to embed a scripting language, but I had a few specific things in mind when I built this:

  • Let users write scripts for extending big complicated products without needing lots of custom code changes or feature flags
  • To be used as a simple game engine scripting language, for running in a browser based editor
  • A way to build custom logic blocks for low-code app environments
SmolScript was created as a learning project. Everything I know about compilers was learned from the amazing book Crafting Interpreters, so if you're also interested in learning how programming languages are created, that is the book for you. It's even free to read online!
